Development Setup

Follow these steps to set up the development environment for the Beetrap Minecraft mod:

Prerequisites

  • Java Development Kit (JDK) 17 or higher
  • Minecraft 1.21.4
  • Fabric Loader
  • IntelliJ IDEA or Eclipse (recommended)
  • Git

Clone and Build

# Clone the repository git clone https://github.com/your-username/beetrap-fabricmc-1.21.4.git cd beetrap-fabricmc-1.21.4 # Build the project ./gradlew build # Run the development client ./gradlew runClient # Run the development server ./gradlew runServer

Agent System

The mod features three types of agents:

  • Physical Agents: Spawn as bee entities in the world with AI capabilities
  • Chat-Only Agents: Communicate through text chat without physical presence
  • Empty Agents: Minimal implementation for testing and development
Note: AI features require an OpenAI API key. Configure this in your environment or configuration files.

Contributing

Code Style

  • Follow Java naming conventions
  • Use meaningful variable and method names
  • Add comments for complex logic
  • Keep methods focused and concise

Testing

# Run all tests ./gradlew test # Run specific test class ./gradlew test --tests "beetrap.btfmc.YourTestClass"

Networking Protocol

The mod uses custom packets for client-server communication:

  • S2C (Server to Client): ShowTextScreenS2CPayload, EntityPositionUpdateS2CPayload
  • C2S (Client to Server): PlayerPollinateC2SPayload, MultipleChoiceAnswerC2SPayload

Adding New Features

  1. Create a new branch for your feature
  2. Implement the feature following the existing architecture
  3. Add appropriate tests
  4. Update documentation if needed
  5. Submit a pull request

Debugging

The mod includes comprehensive logging. Check the logs in:

run/beetrap/logs/beetrap-fabricmc-[timestamp].log

OpenAI Integration

To use AI features, you'll need to:

  1. Obtain an OpenAI API key
  2. Configure the key in your environment
  3. Ensure proper error handling for API calls
  4. Respect rate limits and usage guidelines
